研修用のgithubにアクセスする方法

1.研修用のディレクトリに移動

	自宅PC
cd F:/
cd .apple


	社用
cd C:/
cd appli
cd projects



cd ..	ひとつもどる




	リモートリポジトリのコピー（データもってくる）
git clone https://github.com/tera-25/rookies.git	自分のGITHUBのフォーク先

git clone https://github.com/tera-25/github-test-001.git	；データ送信用


cd rookies			:ここまで入って作業する
cd github-test-001		：データ送信用




	GitHub の最新ブランチ情報を全部取得（重要）
git fetch --all

	GitHubの最新の状態をいまのプロジェクトに反映反映
git reset --hard origin/feature/user-detail/block1


https://github.com/tera-25/rookies.git



2.ブランチを作って研修（大切）
git branch　名前は何でもいい

	研修用ブランチ名一覧
	1. ユーザ情報（詳細）
	feature/user-detail	作成済み

	2. ユーザ登録
	feature/user-create	作成済み

	3. ユーザ削除
	feature/user-delete

	4. ユーザ更新
	feature/user-update

	5. ユーザ一覧
	feature/user-list	


作成したブランチに移動
git checkout xxx




--------ブランチに切り替えた時点で、もうプロジェクトを編集してOK-------

/c/appli/projects/rookies (ブランチ名）→ この状態なら編集開始してOK。


	作業終了後----------------------------------------------------



3.状態確認
git status


4.変更をステージ(高橋さんが言うコミットするための台に乗せる)
git add .		全部やってもいいと思う


コミット
git commit -m "メッセージ"	なんの作業をしたか書く


これが基本ルーティン。
	------１つのブロックの操作が終わったら-----------
作業が確定したらプッシュ
git push

（初回だけ -u が必要）
git push -u origin feature/user-detail/block1




コミット一覧を参照
git log --graph --oneline

コミットのぱすが取得できる
例：ｘｘｘｘ


その他

コミット ｘｘｘｘ に移動	git checkout ｘｘｘｘ

今の変更を退避してから移動	git stash 
				git checkout ｘｘｘｘ

いまの変更を破棄して移動	git restore .
				git checkout ｘｘｘｘ

今の変更をコミットしてから移動	git add .
				git commit -m "WIP: 作業途中"
				git checkout 






今のコミットを特定のブランチに push

git push origin HEAD:




ローカル main を最新に合わせる（上書き）
ローカルで変更がないならこれが最短：

コード
git checkout main
git pull origin main
